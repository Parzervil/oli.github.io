## 一、核心设计哲学

### 1.1 双层肌肉记忆架构
**"基础素质决定下限，技术熟练度决定上限"**

采用**慢变量（基础属性）**与**快变量（技术熟练度）**的双层养成：
- **基础五维（PHY/MOV/TEC/TAC/MEN）**：通过长期训练提升，局间不衰减，构成选手的"天赋底盘"
- **技术熟练度（Muscle Memory）**：具体技术动作的熟练程度（0-100），比赛消耗、训练恢复，体现"千次挥拍"的精度差异

### 1.2 非对称游戏体验
- **70%时间：确定性养成**（训练、属性提升、技术打磨）——符合羽毛球长期主义
- **30%时间：概率性博弈**（关键分决策、ZONE开启、心理波动）——体现竞技不确定性

### 1.3 V3.1安全基石（不可变红线）
严格遵循P0级数值安全：
1. **乘区加法聚合 + 硬封顶1.5x**：Layer1(15%) + Layer2(20%) + Layer3(15%)，绝对禁止真连乘
2. **体力系数全局统一**：`get_sta_coeff(STA) = (STA/100)^0.4 × (0.85 if STA<30 else 1)`，CA与Threat强制复用
3. **动态防御系数1.3-1.7**：弱者保护机制，确保CA差值过大时仍有34.5%救球率

## 二、五维属性与打法演化系统

### 2.1 基础五维定义（V3.1标准）

| 维度 | 英文 | 训练项目 | 对CA的权重贡献 | 养成特点 |
|------|------|----------|---------------|----------|
| **身体素质** | PHY | 力量/爆发力/核心训练 | 进攻型35%/防守型20% | 提升慢，影响STA上限与恢复速度 |
| **移动能力** | MOV | 步伐/敏捷/耐力训练 | 防守型35%/技巧型25% | 影响场地覆盖与Defense_Base |
| **技术水平** | TEC | 手法/控球/动作规范 | 技巧型40%/进攻型25% | 决定Threat值的基础系数 |
| **战术意识** | TAC | 球路/阅读/节奏控制 | 均衡分布 | 影响关键分决策时的信息丰富度与预判成功率 |
| **心理素质** | MEN | 专注/抗压/关键分训练 | 固定5% | 影响FOC上限、压力惩罚减免与ZONE开启阈值 |

**MEN修正机制**：显示值 = 真实值 + 10（确保关键分差异感知）

### 2.2 打法倾向动态演化（非固定标签）

**取消初始固定打法**，改为**训练倾向累计**：
```python
# 进攻指数计算
offense_index = (PHY × 0.3 + TEC × 0.4) - (MOV × 0.3 + TAC × 0.2)

# 阈值触发
if offense_index > 30: 
    倾向 = "进攻型"
    CA计算中PHY权重+5%, TEC权重+3%
    技术熟练度训练：杀球线效率+10%，网前线效率-5%
    
if offense_index > 50:
    进入"固化期"：其他倾向训练效率-20%（模拟打法定型）
```

**打法倾向影响**：
- **加成阶段**：达到阈值后对应属性在CA计算中获得额外权重加成
- **技术解锁**：特定高阶技术需对应倾向值（如"点杀"需进攻指数>40）
- **固化惩罚**：单一倾向超过50时，其他技术训练效率下降（专精代价）

## 三、深度训练系统：肌肉记忆曲线

### 3.1 双层训练架构

#### A. 基础五维训练（慢变量）
- **周期**：每周可分配训练点（随段位提升增加）
- **边际递减**：同一属性连续训练时，收益 = 基础值 × (0.9^n)
- **V3.1体力消耗**：每次训练消耗STA（影响下周初始状态）

| 训练强度 | STA消耗 | 属性收益 | 受伤风险 | 说明 |
|---------|---------|----------|----------|------|
| 恢复性 | 5 | 60%标准值 | 0% | 赛后恢复使用 |
| 标准 | 10 | 100%标准值 | 5% | 常规提升 |
| 超负荷 | 18 | 150%标准值 | 25% | 受伤后PHY-3，强制休息2周 |

#### B. 技术熟练度训练（快变量）

**技术树结构**（杀球线示例）：
```
杀球技术线
├── 基础高远球（熟练度0-40，无门槛）
├── 劈杀（熟练度30-70，需TEC≥60）
├── 跳杀（熟练度50-90，需PHY≥70）
└── 点杀（熟练度70-100，需TEC≥80且进攻倾向>40）
```

**肌肉记忆曲线算法**（V3.1软上限机制应用）：
```python
def training_gain(current_prof, tec_attr):
    """
    current_prof: 当前技术熟练度(0-100)
    tec_attr: 技术水平属性(1-99)
    """
    soft_cap = tec_attr * 1.2  # 技术属性决定熟练度软上限
    
    if current_prof < soft_cap * 0.5:
        return 3.0  # 快速成长期（新手福利）
    elif current_prof < soft_cap * 0.8:
        return 1.5  # 平台期（需大量重复）
    elif current_prof < soft_cap:
        return 0.5  # 瓶颈期（难以突破）
    else:
        return 0.1  # 超水平发挥（极小幅度提升）
```

**顿悟机制**（Layer3词条化）：
- 当技术熟练度卡在瓶颈（如68/70）连续3次训练无提升，第4次有**30%概率**触发"顿悟"
- 顿悟效果：直接+5熟练度，并解锁**技术特性**（如跳杀STA消耗-10%，或劈杀失误率-5%）
- 顿悟获得的特性为**永久被动**，可跨生涯继承（Meta Progression）

**决策计算逻辑（严格遵循V3.1 P0安全）**：
```python
# 1. 基础Threat计算
base_threat = TEC × action_multiplier × proficiency_bonus

# 2. V3.1体力系数（P0安全）
sta_coeff = get_sta_coeff(current_sta)  # (STA/100)^0.4 × (0.85 if STA<30 else 1)

# 3. V3.1 Layer乘区（硬封顶1.5）
layer1 = min(环境加成, 15)  # 主场/气候
layer2 = 20 if is_zone else (15 if is_momentum else 0)  # Zone+20%，气势+15%
layer3 = min(sum(词条加成), 15)  # 大心脏/主场龙等
multiplier = min(1.0 + (layer1+layer2+layer3)/100, 1.5)  # P0硬封顶

# 4. 最终Threat
final_threat = base_threat × sta_coeff × multiplier × random.uniform(0.96, 1.04)

# 5. 防守成功率（V3.1动态系数）
defense_base = MOV × 0.6 + TEC × 0.4
ca_diff = attacker.CA - defender.CA
dynamic_coeff = clamp(1.5 - ca_diff/200, 1.3, 1.7)  # P0安全范围
defense_success = 1 - (final_threat / (defense_base × dynamic_coeff))

# 6. 转换为显示成功率（文档2风格）
display_success = clamp(defense_success × 100, 5%, 95%)
```
